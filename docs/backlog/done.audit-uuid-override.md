## Notes
uuid 9→11 keeps the `v4` API used by gaxios/teeny-request unchanged; the
override only affects the transitive chain. Remove the override once
@google-cloud/storage ships with gaxios@7 / teeny-request@10 (which depend on
patched uuid) and firebase-admin picks it up — recheck during the
firebase-functions 4→7 migration (`todo.dep-major-migrations.md`).

**Pre-existing breakage found during Verify (out of scope here):** main's
lockfile pins firebase-admin@14, which dropped the namespaced API, but
`functions/index.js` still calls `admin.firestore()` etc. (14 uses) — the
emulator fails to load definitions with `TypeError: admin.firestore is not a
function`, so functions cannot deploy from a clean checkout. Needs its own
fix spec (migrate to `firebase-admin/firestore` modular imports or pin
admin@13 until the v1→v2 migration).
